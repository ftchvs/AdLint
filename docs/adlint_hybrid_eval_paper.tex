\noindent\begin{minipage}{\linewidth}
{\sffamily\bfseries\small\color{adlintnavy}Abstract}\par
\vspace{0.3em}
AdLint is a local-first preflight system for ad, landing-page, privacy, and
brand-safety review. This paper reports the current evaluation evidence for a
hybrid architecture in which deterministic policy-as-code rules remain the
decision backbone and a local Ollama model contributes additive review signal.
On a 200-example synthetic rule benchmark, rules reached 1.000 decision
accuracy with zero policy false negatives and eight policy-label false-positive
review notes. On a real local-model smoke run over three seed rows, the
configured model returned status \texttt{ok} for all model-required calls.
Model-only accuracy was 0.667, while hybrid accuracy remained 1.000 and added
three review-only model findings. The result supports delaying fine-tuning and
prioritizing measurement of where the local model adds, rescues, or harms the
deterministic baseline.
A new 13-row public-source real-case diagnostic set adds sourced FTC, ASA/CAP,
and DOJ/HUD-style cases. It is not yet a reliability benchmark, but it exposes
policy-level false-positive and false-negative notes on high-risk public
patterns.
\end{minipage}

\section{Recommendation}

AdLint should focus on a hybrid policy engine rather than a model-only
classifier. The operational rule is simple:

\begin{quote}
\sffamily\small
\textbf{Rules make auditable launch decisions.} The local model adds semantic
review, rewrite assistance, and edge-case evidence, but does not replace
deterministic policy checks.
\end{quote}

This framing has three advantages. First, every blocking or review decision can
trace to policy identifiers, evidence spans, severity, and recommended actions.
Second, the rule path can be regression-tested quickly and reproducibly. Third,
the local model can still provide private, offline reasoning where it is
valuable: ambiguous claims, landing-page context, rewrite suggestions, and
human-readable summaries.

\section{Evaluation Method}

The evaluation now has three layers:

\begin{enumerate}
  \item A 200-example deterministic benchmark,
  \texttt{rule\_benchmark\_v1.jsonl}, used for fast policy-engine regression
  testing.
  \item A required local-model smoke run, \texttt{make model-smoke}, used to
  prove Ollama availability and quantify early hybrid impact on a small subset.
  \item A 13-example real-case diagnostic set,
  \texttt{real\_cases\_v1.jsonl}, derived from public FTC, ASA/CAP, and
  DOJ/HUD-style cases and stored as paraphrased deterministic rows with source
  metadata.
\end{enumerate}

The all-modes runner compares rule-only, model-only, and hybrid outputs. The
new hybrid-value section measures decision improvements, regressions, model
availability, model-only undercalls, model-added findings, model-added
false-positive policy hits, and model-rescued policy false negatives.

\section{Interpretation}

The current data supports four product choices.

\begin{enumerate}
  \item Keep rules as the decision backbone. They are auditable, fast, and
  currently have the strongest measured behavior.
  \item Use the local model where it adds value: semantic concern summaries,
  landing-page context review, and rewrite assistance.
  \item Quantify the local model continuously. The model currently adds review
  findings but has not yet rescued missed policy ids in the smoke subset.
  \item Delay fine-tuning. The measured bottleneck is not a lack of adapter
  training; it is the need for more reviewed examples and clearer model-value
  labels.
  \item Grow real cases before claiming reliability. The new public-source set
  is useful for finding failures, but it is too small and too high-risk-heavy
  to support population-level rates.
\end{enumerate}

\section{Limitations}

The 200-example rule benchmark is synthetic and policy-author authored. It is
useful for regression coverage, not for claiming platform approval or legal
compliance. The real local-model smoke run has only three rows and should be
expanded before making model-quality claims. The 13-row real-case diagnostic
set is sourced, but it is still hand selected and currently contains only
high-risk rows. Current labels do not treat \texttt{model\_policy\_review} as
an expected policy id, so model-added review notes appear as false-positive
policy hits even when they may be useful for human review.

\section{Next Work}

The highest-value next implementation is not fine-tuning. It is a stronger
hybrid eval set with human-reviewed labels for model-added value:

\begin{itemize}
  \item Add rows where rules intentionally miss semantic or landing-page risk.
  \item Label whether \texttt{model\_policy\_review} is useful, noisy, or
  harmful.
  \item Track model-caused review burden separately from true policy false
  positives.
  \item Expand sourced real cases to 50--100 rows with approved, needs-review,
  and high-risk examples before using percentages as reliability evidence.
  \item Only consider LoRA or adapter training after this hybrid baseline shows
  stable gaps that prompting and policy design cannot solve.
\end{itemize}
